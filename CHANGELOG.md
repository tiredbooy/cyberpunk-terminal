# Changelog

All notable changes to this project are documented here.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

This release turns the locked single-palette setup into a themeable, more
portable, and better-documented terminal kit.

### Added

- **Theme engine.** A pure-shell theme switcher built on
  `~/.config/cyberpunk/palette.sh`. Themes live at
  `theme/themes/<name>.sh` and ship with four presets: `neon` (the original
  locked palette), `synthwave`, `matrix`, and `tokyo-neon`. Switching a theme
  regenerates `kitty-theme.conf`, `palette.json`, and the active starship
  palette.
- **`cyber theme [name|list|next|demo]`** — list/switch/cycle/preview themes,
  with live application to a running kitty via `kitty @ set-colors`.
- **`cyber session [name]`** — fzf-pick or launch a named kitty session from
  `~/.config/kitty/sessions/`.
- **`cyber doctor`** — a neon health checklist: truecolor + gradient bar, Nerd
  Font glyph sample, kitty/version/$TERM/$SHELL checks, and per-tool
  present/missing status with the feature each one gates.
- **New kitty maps** for hyperlinked-grep overlay, opening scrollback / last
  command output in nvim, an `ssh` kitten launcher, and broadcast-to-all-windows.
- **`kitty-theme.conf` include** — kitty colors are now generated by
  `cyber theme`; the shipped `theme/kitty-theme.conf` is the default.
- **Tab bar theming** — `kitty/tab_bar.py` reads `palette.json` when present and
  falls back to its hardcoded literals.
- **New kitty sessions** — `ops.session`, `fullstack.session`,
  `writing.session` alongside the existing `dev.session`.
- **Starship palettes** — `[palettes.neon]`, `[palettes.synthwave]`,
  `[palettes.matrix]`, `[palettes.tokyo-neon]` with a `palette = "neon"` switch.
- **Homebrew shellenv** guard for macOS / Apple Silicon (no-op on Linux).
- **Local override** — `~/.config/cyberpunk/local.zsh`, sourced near the end of
  `~/.zshrc`; created only if absent and never clobbered.
- **Optional SSH prompt badge** gated by `CYBERPUNK_SSH_BADGE`.
- **`bootstrap.sh`** — a `curl | bash` entry point that clones/updates the repo
  and runs the installer.
- **tmux config** — `tmux/cyberpunk.tmux.conf` mirrors the palette so the look
  survives on remote/headless hosts.
- **Project meta** — `LICENSE` (MIT), `CONTRIBUTING.md`, this changelog,
  `.editorconfig`, and a GitHub Actions CI pipeline (shellcheck, `bash -n`,
  `zsh -n`, `py_compile`, JSON/TOML validation, and an Ubuntu install
  smoke-test).
- **VHS demo** — `demo/demo.tape` and `demo/README.md` for recording a preview.
- **Rendered promo art** — `screenshots/hero.png` and `screenshots/themes.png`,
  generated from the real palettes, ASCII logo, and prompt by
  `screenshots/render.py`; the pre-modernization static screenshot was removed.

### Changed

- **`cyber update`** now performs a real update: `git pull` the repo, re-run the
  installer non-interactively, and show the new commits.
- **`cyber help`** documents the new theme/session/doctor subcommands and
  keybinds.
- **Git prompt performance** — the stash check folds into a single git
  porcelain pass and `wc`/`tr` pipelines are replaced with zsh builtins, so the
  segment is fork-lean and rendered fresh each prompt (no stale state). A
  background-refresh path for very large repos is reserved for
  `CYBERPUNK_GIT_ASYNC`.
- **Installer** now deploys theme presets, `kitty-theme.conf`, `palette.json`,
  and all `*.session` files (glob), adds `ripgrep` to the tool set, and prints
  next steps suggesting `cyber doctor`.

### Fixed

- **pacman partial-upgrade hazard** — the installer no longer runs `pacman -Sy`
  on its own; it relies on `pacman -S --needed` and advises running
  `sudo pacman -Syu` first.
- **Completion cache logic** — `compinit` now runs a full init (with the security
  check) when the dump is missing or stale, and `compinit -C` only when a fresh
  dump exists; the previous test was inverted and skipped writing a dump on first
  run.
- **Stale git prompt** — removed the mtime-based git-segment cache, which kept
  showing the previous clean/dirty state after a plain working-tree edit (those
  never touch `.git/index` or `HEAD`).
- **Uninstall theme restore** — theme presets are restored/removed per file to
  match how the installer backs them up (by basename), so a non-`--purge`
  uninstall recovers a user's pre-existing `themes/` files.

### Security

- The uninstaller preserves `~/.config/cyberpunk/local.zsh` and
  `~/.config/cyberpunk/theme` (user data) even on `--purge`.
